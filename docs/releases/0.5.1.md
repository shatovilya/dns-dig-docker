# Release 0.5.1 — Configurable host port

**Date:** 2026-07-10  
**App version:** `0.5.1` (`app/main.py`)

## Summary

Adds `DNS_DEBUG_HOST_PORT` to configure the host-side port published by `docker-compose`. Container port remains `8080`; default behavior is unchanged.

## Breaking changes / compatibility notes

- **No breaking changes.** Default mapping remains `8080:8080`.
- Prometheus scrape target (`dns-debugger:8080`) and internal Docker network are unaffected.
- `PORT` inside the container stays `8080`; `DNS_DEBUG_HOST_PORT` is compose-only.

## Configuration notes / env changes

| Variable | Default | Purpose |
|----------|---------|---------|
| `DNS_DEBUG_HOST_PORT` | `8080` | Host port in `docker-compose` `ports` mapping |

Example:

```env
DNS_DEBUG_HOST_PORT=18080
```

```bash
docker compose up -d --build
curl http://localhost:18080/health
```

## Known limitations / follow-ups

- `app/Dockerfile` still binds uvicorn to fixed port `8080`; changing in-container `PORT` without Dockerfile changes is not supported in this release.
