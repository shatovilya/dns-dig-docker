# DNS Debug v0.3.0 — Web UI Observability Layer

**Release date:** 2026-07-10  
**Semver:** minor (0.2.x → 0.3.0)  
**Tag:** `v0.3.0`

## Summary

v0.3.0 adds an optional Web UI observability layer for developers, SREs, QA engineers, and analysts. The dashboard runs on the same FastAPI process, gated by `DNS_DEBUG_UI_ENABLED` (default `false`). Core DNS engine, metrics, and API remain unchanged when UI is disabled.

Key capabilities:

- **Live mode** — auto-refresh, KPI trends, 3-tier information architecture
- **Historical mode** — snapshot-backed time ranges with retention honesty
- **Compare mode** — server-side deltas via `/api/ui/compare`
- **Snapshot persistence** — test completion snapshots for historical analysis
- **Pre-release UX workflow** — 5-stage QA/UX gating for future UI changes

## UX changes

### View modes

| Mode | Behavior |
|------|----------|
| **Live** | Auto-refresh toggle (default ON), KPI ▲/▼ trends vs previous poll, live window presets (15m/1h), `data_source: live_memory` |
| **Historical** | Snapshot picker or `from`/`to` range, auto-refresh off, stale/truncation/retention banners |
| **Compare** | Baseline vs comparison (time ranges, snapshots, per-side test/mode), delta KPI row via server-side `/api/ui/compare` |

### Information architecture

- **3-tier layout:** Status (`zone-status`) → Diagnostics (`zone-diagnostics`) → Drilldown (`zone-drilldown`)
- Sticky sub-nav: **Status | Diagnostics | Drilldown** with scroll-spy (`IntersectionObserver`)
- `global_status` rollup in L1 with diagnosis signals

### Interaction and polish

- Dark/light theme toggle (persisted in `localStorage`)
- Global filters: `test_id`, `resolve_mode`, `query_type`, quick FQDN search, client-side status filter
- Active filter chips, collapsible "History ▾" secondary controls
- Per-panel loading skeletons, empty/error/stale states, panel badges
- KPI click → scroll to target panel
- Events modal (`/api/ui/events`) and diagnosis modal (core API)
- Keyboard support for mode toggle (arrow keys), ARIA on mode buttons
- `ResizeObserver` for dynamic sticky header height (`--top-bar-height`)

### Dashboard panels (10)

Overview, DNS Latency, EDNS, Errors (incl. resolver×error matrix), Garbage, Cache, Records, Load, MTR, Rankings.

## Responsive / adaptive fixes

Breakpoints in `dashboard.css`: **1440px, 1366px, 1279px, 1023px, 768px, 640px, 390px**.

| Fix | Detail |
|-----|--------|
| Laptop sticky header overlap | Fixed overlap at 1024–1440px widths — core triage flow intact |
| Chart legend reflow | Legend position adapts at 768px via `matchMedia` in JS |
| KPI grid reflow | Grid columns adjust across breakpoints |
| Filter bar compaction | Horizontal chip scroll on mobile; filter bar wraps at laptop widths |
| Chart lifecycle | Destroy/recreate charts on mode switch (prevents stale Chart.js instances) |

## Localization / Russian language support

**None in this release.**

- UI is English-only (`<html lang="en">` in `dashboard.html`)
- No locale files, translation bundles, or i18n libraries
- Russian or other language support is a future feature, not part of v0.3.0

## QA / UX workflow changes

This release introduces the **5-stage pre-release UX workflow** for all visual/behavioral UI changes:

| Stage | Role | Purpose |
|-------|------|---------|
| 1. Self-check | DNS engineer | Feature completeness, states, responsive basics |
| 2. UX review | UX designer | Pre-release UX audit deliverable |
| 3. QA review | QA engineer | Acceptance, responsive widths, state coverage |
| 4. Fix pass | DNS engineer | Close P0/P1 findings |
| 5. Release readiness | All roles | No blockers; docs synced |

**New agent skills:**

- [`.ai/skills/qa-ui/SKILL.md`](../../.ai/skills/qa-ui/SKILL.md) — acceptance, regression, API↔UI consistency
- [`.ai/skills/ux-designer/SKILL.md`](../../.ai/skills/ux-designer/SKILL.md) — dashboard IA, states, pre-release UX audit

**Release blockers (P0/P1):** wrong compare/historical data, security regression, misleading cache copy, laptop/tablet layout breakage, missing state coverage, skipped UX audit or QA pass, stale docs.

## Breaking changes / compatibility

**None.**

- Web UI is opt-in via `DNS_DEBUG_UI_ENABLED=false` (default)
- Core API paths and Prometheus metric names unchanged
- UI JSON contracts are additive only
- `DNS_DEBUG_UI_READONLY=true` (default) — no mutating actions from browser
- Existing deployments without UI env vars continue to work unchanged

## Configuration notes / env changes

### Web UI

| Variable | Default | Notes |
|----------|---------|-------|
| `DNS_DEBUG_UI_ENABLED` | `false` | Master feature flag |
| `DNS_DEBUG_UI_BASE_PATH` | `/dns-debug` | Dashboard + API base path |
| `DNS_DEBUG_UI_READONLY` | `true` | View-only UI |
| `DNS_DEBUG_UI_REFRESH_SECONDS` | `5` | Live-mode polling interval |
| `DNS_DEBUG_UI_AUTH_ENABLED` | `true` | Auth on UI routes when UI enabled |
| `DNS_DEBUG_UI_ALLOWED_ROLES` | `read-only,operator,admin` | Roles for UI access |
| `DNS_DEBUG_UI_IP_ALLOWLIST` | `[]` | Optional IP restriction |
| `DNS_DEBUG_UI_CSP` | (see `.env.example`) | CSP for Chart.js CDN |

### Snapshot persistence (historical mode)

| Variable | Default | Notes |
|----------|---------|-------|
| `SNAPSHOT_ENABLED` | `true` | Persist snapshots on test completion |
| `SNAPSHOT_DIR` | `data/snapshots` | Snapshot storage directory |
| `SNAPSHOT_RETENTION_COUNT` | `20` | Max snapshots kept |

### Docker

Add volume mount for snapshot persistence across restarts:

```yaml
volumes:
  - ./data/snapshots:/app/data/snapshots
```

### Enable UI locally

```env
DNS_DEBUG_UI_ENABLED=true
DNS_DEBUG_UI_BASE_PATH=/dns-debug
API_AUTH_ENABLED=false
```

Open: `http://localhost:8080/dns-debug/`

## Known limitations / follow-ups

| Limitation | Detail |
|------------|--------|
| Cache heuristic | `dns_debug_possible_cached_response_total` and cache UI cards are latency-based heuristics — not real Docker DNS cache introspection |
| English-only UI | No i18n; all strings hardcoded in templates/JS |
| Historical mode | Depends on snapshot retention (`SNAPSHOT_RETENTION_COUNT`); truncated event buffer shows honesty banners |
| Compare mode | Requires baseline data on both sides; zero-baseline periods show null deltas |
| MTR panel | Empty state when `MTR_ENABLED=false` |
| EDNS panel | Note visible when only edns0 has data |
| Separate UI port | `DNS_DEBUG_UI_PORT` documented but recommended pattern is same process on 8080 |
