# Release 0.5.2 — Compose resource limits

**Date:** 2026-07-10  
**App version:** `0.5.2` (`app/main.py`)

## Summary

Adds `deploy.resources.limits` to `docker-compose.yml` for all services. Default limits cap host resource usage while keeping dev/small deployments predictable.

## UX changes

None.

## Responsive / adaptive fixes

None.

## Localization / Russian language support

None.

## QA / UX workflow changes

None.

## Documentation / rules / skills / agent updates

- `README.md` — default resource limits table and tuning note for autonomous/high-RPS workloads

## Breaking changes / compatibility notes

- **No breaking changes.** Limits apply at container runtime only; API, metrics, and internal Docker networking are unchanged.
- Under heavy autonomous load, 512M / 1 CPU on `dns-debugger` may be tight — raise limits in `docker-compose.yml` as needed.

## Configuration notes / env changes

No new env vars. Limits are set in `docker-compose.yml`:

| Service | Memory | CPU |
|---------|--------|-----|
| `dns-debugger` | 512M | 1.0 |
| `postgres` | 256M | 0.25 |
| `prometheus` (monitoring profile) | 256M | 0.25 |

Requires Docker Compose v2.23+ for local `deploy.resources.limits` enforcement (verify with `docker stats`).

## Known limitations / follow-ups

- Older `docker compose` versions may ignore `deploy.resources` outside Swarm; use `mem_limit` / `cpus` fallback if limits are not applied.
